KEATING PROJECT TEST REFORM CONTEXT

Goal: Rewrite the test suite using fast-check (property-based testing) and Antithesis guidelines.

Antithesis Key Principles:
- Test PROPERTIES not sequences — "always true", "never true", "sometimes true"
- Deterministic simulation: controlled entropy, reproducible seeds
- Validate continuously: work → validate → work
- State space exploration: diverse inputs + system faults
- Always properties: disproved by single counterexample
- Sometimes properties: proved by single example
- Reliability properties: invariants, determinism/purity, differential testing, ACID
- Swarm testing: randomly enable/disable features
- Buggification: make rare events common
- Cover the entire surface including config, admin, startup, error paths
